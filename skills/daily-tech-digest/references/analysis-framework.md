# Analysis Framework

技术趋势分析框架，用于生成结构化洞察。

## 领域分类

| 领域 | 包含内容 | 关注维度 |
|------|---------|---------|
| AI/ML | 模型发布、Agent、工具链、应用 | 能力突破、可用性、成本 |
| DevTools | IDE、CLI、CI/CD、测试、监控 | 效率提升、开发体验 |
| Infrastructure | 云服务、数据库、网络、安全 | 稳定性、性能、成本 |
| Product | 新产品、功能更新、UX 创新 | 用户价值、商业模式 |
| Business | 融资、收购、战略、政策 | 市场格局、竞争态势 |
| Open Source | 新项目、重大更新、社区动态 | 采用率、生态影响 |

## 信号强度判断

| 强度 | 标准 |
|------|------|
| 🔴 强 | 多源同时出现；热度 > 500 points；头部公司/人物发声 |
| 🟡 中 | 2个以上来源提及；热度 200-500；值得持续关注 |
| 🟢 弱 | 单一来源；热度 < 200；可能是噪音或早期信号 |

## 趋势阶段判断

| 阶段 | 特征 |
|------|------|
| 新兴 | 首次出现或近期突然升温；讨论以「这是什么」为主 |
| 成长 | 持续热度；讨论以「如何使用」为主；出现落地案例 |
| 成熟 | 已成为标配；讨论以「最佳实践」「对比选型」为主 |
| 衰退 | 热度下降；讨论以「替代方案」「局限性」为主 |

## 影响范围判断

| 范围 | 特征 |
|------|------|
| 广泛 | 影响整个行业或多个领域 |
| 垂直 | 影响特定技术栈或行业 |
| 小众 | 仅影响特定场景或人群 |

## 洞察提炼模板

对于每个重要趋势，回答以下问题：

1. **What（现象）**: 用一句话描述观察到的现象
2. **Why（本质）**: 这个现象背后的驱动力是什么？技术突破？市场需求？政策变化？
3. **So What（影响）**: 对开发者/企业/用户意味着什么？
4. **What's Next（预判）**: 接下来可能会怎样发展？

## 输出语言风格

- 技术术语保留英文原文
- 摘要简洁，洞察深入
- 避免泛泛而谈
- 数据支撑观点
- 保持客观中立
